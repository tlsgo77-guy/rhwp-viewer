# rhwp 단축키 정책 문서

**작성일**: 2026-04-13
**관련 이슈**: [#131](https://github.com/edwardkim/rhwp/issues/131)

---

## 1. 기본 원칙

### 1.1 한컴 호환 우선

한글과컴퓨터 한/글의 단축키를 최대한 호환한다. 사용자가 한/글 습관 그대로 rhwp를 사용할 수 있어야 한다.

### 1.2 브라우저 충돌 처리

브라우저/운영체제가 단축키를 선점하여 편집 영역에 전달하지 않는 경우, `Alt+Shift+*` 계열로 재매핑한다.

**재매핑 원칙**:
- `Ctrl+Shift+*` 중 브라우저가 양보하지 않는 키 → `Alt+Shift+*`로 변경
- 재매핑 시 기존 `Alt+Shift+*` 사용 키와 충돌 여부 반드시 확인
- 한컴 원본 키는 주석으로 보존

### 1.3 표 조작 단축키 불변

표 편집 모드 전용 단축키(`F5`, `Tab`, `Alt+방향키` 등)는 변경하지 않는다.

## 3. Chord 단축키 (2단계 입력)

| 1번째 키 | 2번째 키 | 기능 |
|---------|---------|------|
| `Ctrl+G` | `C` / `ㅊ` | 조판 부호 보이기/숨기기 |
| `Ctrl+G` | `T` / `ㅅ` | 문단 부호 보이기/숨기기 |
| `Ctrl+G` | `P` / `ㅍ` | 화면 확대 — 쪽 맞춤 |
| `Ctrl+G` | `W` / `ㅈ` | 화면 확대 — 폭 맞춤 |
| `Ctrl+G` | `Q` / `ㅂ` | 화면 확대 — 100% |
| `Ctrl+K` | `B` / `ㅠ` | 책갈피 삽입 |
| `Ctrl+K` | `N` / `ㅜ` | 문단 번호 모양 |
| `Ctrl+N` | `N` / `ㅜ` | 각주 삽입 |
| `Ctrl+N` | `S` / `ㄴ` | 쪽 감추기 |
| `Alt+V`  | `T` / `ㅅ` | 투명 선 보이기/숨기기 |

## 4. 전역 단축키 (문서 미로드 상태에서도 동작)

| 단축키 | 기능 | 처리 위치 |
|--------|------|----------|
| `Alt+N` | 새 문서 | `setupGlobalShortcuts()` (main.ts) |

편집 영역이 활성 상태(`InputHandler.active = true`)가 아니어도 동작한다.

---

## 5. 커맨드 팔레트

| 단축키 | 기능 |
|--------|------|
| `Ctrl+/` | 커맨드 팔레트 열기 |

편집 영역 활성 상태에서만 동작. 등록된 모든 커맨드(152개)를 한글/영문으로 검색하여 실행할 수 있다.

---

## 6. 한글 IME 이중 매핑

모든 영문 단축키는 한글 IME 입력 상태에서도 동작하도록 한글 자모 키를 이중 등록한다.

| 영문 키 | 한글 IME 키 |
|--------|------------|
| `n` | `ㅜ` |
| `g` | `ㅎ` |
| `k` | `ㅏ` |
| `v` | `ㅍ` |
| `t` (Chord 2번째) | `ㅅ` |
| `c` (Chord 2번째) | `ㅊ` |
| `p` (Chord 2번째) | `ㅍ` |
| `w` (Chord 2번째) | `ㅈ` |
| `q` (Chord 2번째) | `ㅂ` |

---

## 7. Shift+Enter

강제 줄 나누기(문단 유지, 줄만 바꿈). `input-handler-keyboard.ts`의 `case 'Enter'`에서 `e.shiftKey` 분기로 처리. 별도 단축키 테이블 등록 없이 직접 구현.
