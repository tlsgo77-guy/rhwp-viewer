## 배경

### 1. 단축키 미구현

`[보기] → [표시/숨기기] → [투명 선]` 기능이 메뉴 클릭으로만 접근 가능하여 불편하다.
한컴오피스의 `Alt+V, T` Chord 조합을 계승하되 브라우저/기존 단축키 충돌을 피한다.

- `Ctrl+F5` — 브라우저 강제 새로고침과 충돌
- `Alt+T` / `Alt+ㅅ` — `format:para-shape`에 사용 중
- `F5` 계열 — 셀 선택 모드와 충돌
- **`Alt+V → T` Chord** — 충돌 없음, 한컴 원본 계승

### 2. 초기값 불일치 버그

`view:border-transparent` 커맨드의 `let showBorders = false` 초기값이 WASM 실제 초기 상태와 불일치.
첫 번째 토글 시 변화 없고 두 번째에 적용되는 버그.

---

## 수정 방향

### 단축키

`input-handler-keyboard.ts`에 `chordMapV` 테이블 추가:
- `Alt+V` 입력 시 Chord 대기 모드 진입
- 이후 `T` (또는 `ㅅ` IME) 입력 시 `view:border-transparent` 커맨드 실행

### 초기값 버그

`view.ts`의 `showBorders` 초기값을 WASM 실제 상태(`getShowTransparentBorders()`)와 동기화하거나,
WASM 초기값을 `false`로 고정하여 일치시킨다.
