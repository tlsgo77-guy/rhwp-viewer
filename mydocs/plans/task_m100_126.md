## 문제 요약

`npm run build` 실행 시 `tsc` 단계에서 3종의 에러가 발생하여 빌드가 실패한다.
`tsconfig.json`은 `strict: true`로 설정되어 있으며, 우회 없이 코드 수정으로 해결한다.

## 에러 및 수정 방향

### 1. TS2341 — `WasmBridge.doc` private 멤버 외부 접근 (2건)

- `src/command/commands/file.ts:136` — `wasm.doc!.renderPageSvg(i)`
- `src/main.ts:532` — `wasm.doc?.renderPageSvg(...)`

`WasmBridge.doc`는 `private`이므로 외부에서 직접 접근 불가.
`WasmBridge`에 이미 `renderPageSvg()` public 메서드가 있는지 확인 후,
없으면 public 위임 메서드를 추가한다.

### 2. TS2551 — `create_empty` → `createEmpty` 오탈자 (1건)

- `src/hwpctl/index.ts:351` — `HwpDocument.create_empty()`

WASM API 메서드명은 `createEmpty`이므로 단순 수정.

### 3. TS2304 — `chrome` 전역 타입 미선언 (3건)

- `src/main.ts:461,466` — `chrome.runtime.sendMessage`

Chrome 익스텐션 API 타입 패키지 `@types/chrome`을 devDependency에 추가한다.

## 수정 금지 사항

- `tsconfig.json` strict 옵션 완화 금지
- `// @ts-ignore` / `// @ts-expect-error` 우회 금지
- `any` 타입 캐스팅 우회 금지

## 완료 기준

`npm run build` (tsc + vite build) 에러 없이 성공.
