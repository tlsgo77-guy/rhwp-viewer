## 배경

한컴 단축키 체계는 Windows 3.1 시절부터 설계되어 현대 브라우저/OS 예약 키와 충돌이 많다.
CDP로 한컴 공식 단축키를 전수 추출하여 Chrome 예약 키와 비교 분석한 결과:

- **브라우저가 양보하는 키**: `Ctrl+S`, `Ctrl+P`, `Ctrl+B/I/U` 등 — 편집 영역 포커스 시 rhwp가 가로챌 수 있음
- **브라우저가 절대 양보하지 않는 키**: `Ctrl+T`, `F5`, `F11`, `F12`, `Ctrl+Shift+I/J/C/N/T/R` 등 — 재매핑 필요

한컴 웹오피스 포함 기존 제품 모두 이 문제를 해결 못한 채 구형 문서만 유지 중.
rhwp는 충돌 정책을 명확히 설계하고 `/` 커맨드 팔레트로 사용자에게 투명하게 안내한다.

---

## 수정 방향

### 호환 레이어
브라우저가 양보하는 범위에서 한컴 단축키 그대로 지원.

### 재매핑 레이어
브라우저가 절대 양보하지 않는 키는 rhwp 독자 키로 대체.
Chord 방식(`Ctrl+G,*` 등) 적극 활용.

### `/` 커맨드 팔레트
- `/` 입력 → 커맨드 팔레트 팝업
- 기능명 검색 → 단축키 함께 표시
- 재매핑된 키 안내 내장
- 단축키를 몰라도 기능 접근 가능 + 자연스러운 학습

## 작업 범위

### 1단계 — [보기] Ctrl+G 계열 완성
- `chordMapG`에 추가: `t → view:para-mark`, `p → view:zoom-fit-page`, `w → view:zoom-fit-width`, `q → view:zoom-100`
- `index.html` 보기 메뉴 `md-shortcut` 표시 정비

### 2단계 — [서식] 기본 글자 서식 단축키
- `Ctrl+B`, `Ctrl+I`, `Ctrl+U` — 진하게/기울임/밑줄
- `Ctrl+]`, `Ctrl+[` — 글씨 크게/작게
- `Ctrl+Shift+L/R/M` — 왼쪽/오른쪽/양쪽 정렬 (브라우저 충돌 키 재매핑 포함)

### 3단계 — `/` 커맨드 팔레트
- `/` 키 입력 → 커맨드 팔레트 UI
- 전체 커맨드 검색 + 단축키 표시
- 재매핑 안내 포함

### 4단계 — 재매핑 정책 문서화 + 나머지 누락 단축키
- `Shift+Enter` 강제 줄 나누기
- `Alt+C` 모양 복사
- 단축키 정책 문서 (`mydocs/tech/shortcut_policy.md`)
- `index.html` 전체 `md-shortcut` 정비

## 완료 기준
