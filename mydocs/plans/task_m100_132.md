### 구현 방식 결정

**SVG 내보내기**: 두 가지 경로 검토
1. rhwp CLI(`export-svg`) 호출 — Node.js child_process로 네이티브 바이너리 실행
2. WASM을 익스텐션 Node.js 컨텍스트에서 직접 호출

→ **WASM 직접 호출** 채택. CLI 바이너리 의존성 없이 pkg/ WASM으로 처리 가능. webview가 아닌 익스텐션 host에서 실행하므로 파일 I/O 권한 문제 없음.

**인쇄**: webview의 `window.print()` 호출. VS Code webview에서 인쇄 다이얼로그 지원.

## 구현 계획 (단계별)

### 1단계: package.json — commands + menus 등록

4개 커맨드를 탐색기/에디터 탭 컨텍스트 메뉴에 등록한다.

```json
"contributes": {
  "commands": [
    { "command": "rhwp.print",         "title": "HWP: 인쇄" },
    { "command": "rhwp.exportSvg",     "title": "HWP: SVG로 내보내기" },
    { "command": "rhwp.debugOverlay",  "title": "HWP: 디버그 오버레이 보기" },
    { "command": "rhwp.dumpParagraph", "title": "HWP: 문단 덤프" }
  ],
  "menus": {
    "explorer/context": [
      { "command": "rhwp.print",         "when": "resourceExtname == .hwp || resourceExtname == .hwpx", "group": "rhwp@1" },
      { "command": "rhwp.exportSvg",     "when": "resourceExtname == .hwp || resourceExtname == .hwpx", "group": "rhwp@2" },
      { "command": "rhwp.debugOverlay",  "when": "resourceExtname == .hwp || resourceExtname == .hwpx", "group": "rhwp@3" },
      { "command": "rhwp.dumpParagraph", "when": "resourceExtname == .hwp || resourceExtname == .hwpx", "group": "rhwp@4" }
    ],
    "editor/title/context": [
      { "command": "rhwp.print",         "when": "resourceExtname == .hwp || resourceExtname == .hwpx", "group": "rhwp@1" },
      { "command": "rhwp.exportSvg",     "when": "resourceExtname == .hwp || resourceExtname == .hwpx", "group": "rhwp@2" },
      { "command": "rhwp.debugOverlay",  "when": "resourceExtname == .hwp || resourceExtname == .hwpx", "group": "rhwp@3" },
      { "command": "rhwp.dumpParagraph", "when": "resourceExtname == .hwp || resourceExtname == .hwpx", "group": "rhwp@4" }
```

### 2단계: 인쇄 커맨드 구현

- `HwpEditorProvider`에서 열려있는 webview 패널을 추적 (파일 URI → 패널 Map)
- `rhwp.print` 커맨드 → 해당 파일의 webview에 `{ type: 'print' }` 메시지 전송
- `viewer.ts`에서 메시지 수신 → `window.print()` 호출
- webview가 열려있지 않은 경우 → 먼저 열고 로드 완료 후 인쇄

### 3단계: SVG 내보내기 커맨드 구현

- `rhwp.exportSvg` 커맨드 → 출력 폴더 선택 다이얼로그 (기본: 파일과 동일 폴더)
- 익스텐션 host에서 WASM 초기화 → HWP 파일 로드 → 페이지별 `renderPageSvg()` 호출
- SVG 파일 저장: `{파일명}_p{n}.svg`
- 진행 상황 progress notification 표시
- 완료 시 알림 + "폴더 열기" 버튼

### 4단계: 디버그 오버레이 + 문단 덤프 커맨드 구현

**디버그 오버레이 (`rhwp.debugOverlay`)**
- WASM으로 모든 페이지를 `--debug-overlay` 모드로 렌더링 (문단/표 경계 + 인덱스 라벨)
- 결과 SVG를 임시 파일(`/tmp/rhwp-debug-{hash}.svg`)에 저장
- `vscode.open()` → VS Code 내장 SVG 미리보기 탭으로 바로 열기
- 파일 저장 없이 확인만 가능한 개발자용 도구

**문단 덤프 (`rhwp.dumpParagraph`)**
- QuickPick으로 섹션 번호 입력 (기본값: 0)
- QuickPick으로 문단 번호 입력 (기본값: 0)
- WASM으로 해당 문단의 ParaShape / LINE_SEG / 표 속성 조회
- 결과를 Output 채널 `HWP Dump`에 출력 (터미널 없이 VS Code 내에서 확인)
- 출력 형식: CLI `dump` 명령과 동일한 텍스트 포맷

### 5단계: 빌드 + 검증

- webpack 빌드 (`npm run build`)
- VS Code Extension Development Host로 4개 커맨드 동작 확인
- 최종 결과 보고서 작성

## 작업 제약

- 편집 기능은 추가하지 않는다 (뷰어 기능만 확장)
- CLI 바이너리(rhwp) 의존성을 추가하지 않는다
- 기존 커스텀 에디터(`rhwp.hwpViewer`) 동작은 변경하지 않는다
- 디버그 오버레이 / 문단 덤프는 개발자 도구 성격으로, 일반 사용자 UX에 영향을 주지 않는다
