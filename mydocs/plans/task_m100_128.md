## 배경

CodeQL Code Scanning Alert #14 (`js/xss-through-dom`, severity: **high**).

- **위치**: `rhwp-chrome/content-script.js:67` — `img.src = dataUri`
- **원인**: `isSafeImageUrl()`로 프로토콜 검증은 하지만, 원본 DOM 문자열을 그대로 `img.src`에 할당
- CodeQL은 "DOM 원본 값 → img.src" 데이터 흐름을 XSS 위험으로 판단
- 공공기관 보안 감사에서 `high` 경고는 도입 거부 사유가 될 수 있음
- 썸네일 기능은 한컴 대비 킬러 펑션이므로 기능 유지하면서 경고 제거 필요

## 수정 방향

`isSafeImageUrl()` 통과 후 원본 문자열 대신 `new URL(dataUri).href`로 재구성된 값 사용.

- CodeQL이 "외부 입력 → URL 파싱 → `.href` 추출" 흐름을 안전한 변환으로 인식할 것으로 기대
- 기능 동작은 동일 (정규화된 URL이므로)

## 완료 기준

- `content-script.js` 수정 (`img.src = new URL(dataUri).href`)
- `devel` push → CodeQL 분석 결과에서 Alert #14 해소 확인
- 해소되지 않을 경우 추가 접근 방법 검토
