## 3. 현재 구현된 증분 처리 메커니즘

rhwp는 이미 상당 수준의 증분 레이아웃을 보유하고 있다:

### dirty 플래그 시스템

| 메커니즘 | 위치 | 역할 |
|----------|------|------|
| `dirty_sections` | document.rs | 섹션 수준 dirty 플래그 |
| `dirty_paragraphs` | document.rs | 문단 수준 dirty 비트맵 |
| `mark_section_dirty()` | rendering.rs | 섹션만 dirty 마킹 |
| `mark_paragraph_dirty()` | rendering.rs | 특정 문단만 dirty 마킹 |

### 증분 재조합/측정

| 메커니즘 | 역할 |
|----------|------|
| `recompose_paragraph()` | 단일 문단만 재조합 |
| `insert/remove_composed_paragraph()` | 문단 분할/병합 시 항목 삽입/제거 |
| `measure_section_selective()` | dirty 문단만 재측정, 나머지 캐시 재사용 |

### 수렴 루프

| 메커니즘 | 역할 |
|----------|------|
| `para_column_map` 비교 | 다단 컬럼 재배치 감지 및 반복 처리 |
| 최대 3회 반복 | cascade 안정화 |

## 5. 기술 배경 (증분 레이아웃 아키텍처 조사)

5개 시스템의 증분 레이아웃 패턴을 조사했다 (`incremental_layout_research.md`):

| 시스템 | 핵심 전략 | rhwp 적용 가능성 |
|--------|----------|-----------------|
| LibreOffice | 프레임별 dirty 플래그 + 유휴 레이아웃 | 높음 (현재 dirty 시스템과 유사) |
| Typst | 제약 기반 메모이제이션 (comemo) | 중간 (캐시 효율 우수, 구현 복잡) |
| ProseMirror | 불변 문서 + diff | 중간 (편집 모델에 유용) |
| Google Docs | Canvas 자체 엔진, 줄 단위 업데이트 | 참고 |
| xi-editor | 문단 독립 캐싱 + delta 전파 | 높음 (HWP 문단 독립성과 부합) |

### 페이지네이션 안정화 전략

- 변경된 문단부터 **페이지 경계가 안정화될 때까지** 재레이아웃
- 페이지 N의 마지막 항목이 변경 전과 동일하면 → 페이지 N+1 이후 재사용
- "안정화 지점(stabilization point)" 감지가 핵심

### 표 편집 최적화 전파 경로

```
셀 내 텍스트 변경
 → 셀 내 문단 재레이아웃
 → 셀 높이 변경?
    ├─ 아니오 → 종료 (셀 내부만 재렌더링)
    └─ 예 → 행 높이 재계산 → 표 전체 높이 변경? → 페이지 넘침 확인
```

## 7. Issue #2 수행 범위

### 3대 범위

| 범위 | 핵심 | 관련 Gap |
|------|------|---------|
| **문단 추가/삭제** | IR 정합성 유지 + 전후 layout/pagination cascade | Gap 2,3,7 |
| **텍스트 편집** | 줄바꿈 발생 시 이후 layout/pagination 전파 | Gap 1,8 |
| **컨트롤 배치** | 이미지/글상자/도형/표의 배치 속성에 따른 동적 레이아웃 변경 | Gap 6 |

#### 범위 1: 문단 추가/삭제

- Enter로 문단 분할(split) → 새 문단 IR 생성 + 후속 문단 인덱스/vpos 갱신
- Backspace로 문단 병합(merge) → IR 통합 + 후속 문단 위치 재계산
- 페이지/컬럼 브레이크 삽입 → 증분 재조합 (현재 전체 재조합, Gap 7)
- 다단 문서에서 split/merge 시 컬럼 재배치 감지 (Gap 2,3)

#### 범위 2: 텍스트 편집

- 문단 내 텍스트 추가/삭제 → 줄바꿈 위치 변경 → 문단 높이 변경
- 문단 높이 변경 → 후속 문단 vpos cascade → 페이지 넘침 확인
- 다단 문서에서 인접 문단이 컬럼 이동 시 수렴 루프 감지 (Gap 1)
- vpos cascade 후 컬럼 경계 감지 (Gap 8)

#### 범위 3: 컨트롤 배치

- treat_as_char(글자처럼) 표/이미지: 문단 내 인라인 배치, 줄 높이 영향
- 위아래 배치 표/이미지: 문단 간 공간 차지, 후속 문단 위치 변경
- 자리차지 않는 배치(본문 앞/뒤): 레이아웃에 영향 없는 플로팅
- 셀 내 편집 시 composed 즉시 갱신 (Gap 6)
- 글상자 내 텍스트 편집 → 글상자 크기 변경 → 호스트 문단 재조판
